Overview

In this activity we will explore the turtlebot3 and interbotix_arm packages to see how they use xacro, urdf, and sdf files

Exploration of turtlebot 3 urdf

The turtlebot3 is a small differential drive robot designed to work with ROS. We will focus on the "burger" variant.

  1. Explore the project directory roscd turtlebot3_description.
    • This package describes the turtlebot3
    • Of particular interest is the urdf directory
  2. Notice that the description is divided across several files
  3. common_properties.xacro is a central place to define some colors for the urdf file
  4. turtlebot3_burger.urdf.xacro is the basic urdf for rviz.
    • By using .stl files for the link geometry, complex geometry is captured with few links. If you had a CAD model of a robot you would convert to .stl or .dae and use this feature.
    • Notice that the .stl file is visual only. The collision is a bounding box.
    • The inertial values are likely the output of a CAD program
  5. turtlebot3_burger.gazebo.xacro adds gazebo-specific tags to the link.
    • Notice that the Differential Drive plugin is used to control the robot.
    • The turtlebot also has an imu and laser scanner that are modeled using built-in gazebo plugins.

Exploration of Interbotix ROS arms

  1. https://github.com/Interbotix/interbotix_ros_manipulators/tree/main/interbotix_ros_xsarms
  2. Let's examine the interbotix_descriptions package
  3. Often, the gazebo code is completely separated from the robot description, enabling the description to be used without requiring gazebo
  4. Examine the urdf directory
    • Notice how the gazebo code is only included if explicitly requested
    • Notice how the name of everything is prefixed with a robot name, allowing you to have multiple interbotix arms on the same machine
  5. The <transmission> tags are used by the ros_control package
    • Many ROS packages use ros_control as a standard interface for controlling the robot
  6. Look at interbotix_xsarm_gazebo/config/interbotix_texture.gazebo
    • This is setting up materials and some other gazebo specific properties
  7. Look at the gazebo launch file, what is it doing?
